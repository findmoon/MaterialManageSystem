# 物料管理系统 - 部署文档

## 1. 系统概述

物料管理系统是一个基于ASP.NET Core 10.0和Vue 3的现代化物料管理平台，采用分层架构设计，提供物料入库、出库、库存预警、在线监控等核心功能。

### 项目架构

```
├── MaterialManageSystem.Core/      # 核心层 - 实体、接口、DTO
├── MaterialManageSystem.Infrastructure/ # 基础设施层 - 数据访问、服务实现
├── MaterialManageSystem.Api/       # API层 - 控制器、中间件、SignalR
├── MaterialManageSystem.Admin/     # 管理后台 - Razor Pages
└── MaterialManageSystem.Vue/       # 前端 - Vue 3 + TypeScript
```

### 1.1 项目模块说明

| 项目名称 | 层级 | 功能说明 |
|----------|------|----------|
| **MaterialManageSystem.Core** | 核心层 | 定义领域实体、业务接口、数据传输对象(DTO)、枚举类型 |
| **MaterialManageSystem.Infrastructure** | 基础设施层 | 实现数据访问(Repository模式)、预警检测服务、数据库迁移 |
| **MaterialManageSystem.Api** | API层 | 提供RESTful API、JWT认证、SignalR实时推送、中间件处理 |
| **MaterialManageSystem.Admin** | 管理后台 | Razor Pages管理界面，提供基础管理功能 |
| **MaterialManageSystem.Vue** | 前端 | Vue 3单页应用，提供用户交互界面 |

#### 1.1.1 MaterialManageSystem.Core（核心层）

**职责**：定义系统的核心业务模型和接口契约

**核心目录结构**：
```
Core/
├── Entities/           # 领域实体（数据库表映射）
│   ├── BaseEntity.cs   # 实体基类（包含Id、CreateTime、UpdateTime）
│   ├── Warehouse.cs    # 库房实体
│   ├── PartNo.cs       # 物料实体
│   ├── ReelId.cs       # 料卷实体
│   ├── User.cs         # 用户实体
│   ├── WarningRecord.cs # 预警记录实体
│   └── ...
├── DTOs/               # 数据传输对象
│   └── DTOs.cs         # 各类请求/响应DTO定义
├── Enums/              # 枚举类型
│   └── Enums.cs        # 用户类型、料卷状态、预警类型等
└── Interfaces/         # 业务接口
    ├── IRepository.cs          # 通用仓储接口
    ├── IServices.cs            # 业务服务接口
    └── ISpecificRepositories.cs # 各实体专用仓储接口
```

**主要实体说明**：

| 实体 | 说明 | 关键字段 |
|------|------|----------|
| **Warehouse** | 库房信息 | Id, Code(编码), Name(名称), Location(位置) |
| **Rack/Layer/Cell** | 货架/层/仓位 | 层级结构，关联库房 |
| **PartNo** | 物料信息 | PartNo(物料编码), Name, Spec(规格), Unit(单位), WarningQty(预警值) |
| **ReelId** | 料卷信息 | ReelCode(料卷编号), PartNoId, Quantity(数量), Status(状态) |
| **User** | 用户信息 | Username, PasswordHash, UserType(用户类型) |
| **WarningRecord** | 预警记录 | WarningType(预警类型), Status(处理状态), TargetId(关联ID) |

## 4. 部署步骤

### 4.1 数据库部署

1. **创建数据库**

打开SQL Server Management Studio，执行以下SQL脚本创建数据库：

```sql
CREATE DATABASE MaterialManageSystem
GO
USE MaterialManageSystem
GO
```

2. **数据库连接字符串**

修改 `MaterialManageSystem.Api/appsettings.json` 文件：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=你的服务器地址;Database=MaterialManageSystem;User ID=用户名;Password=密码;TrustServerCertificate=True;MultipleActiveResultSets=true"
  }
}
```
Server=127.0.0.1;Database=MaterialWarningDB;User Id=Fujiadmin;Password=admin;TrustServerCertificate=True

3. **执行数据库迁移**

```bash
cd MaterialManageSystem.Api
dotnet add package Microsoft.EntityFrameworkCore.Design

dotnet ef migrations add InitialCreate --project ../MaterialManageSystem.Infrastructure --startup-project .

dotnet ef database update --project ../MaterialManageSystem.Infrastructure --startup-project .
```

### 4.2 后端API部署

#### 方式一：使用Docker部署（推荐）

创建 `Dockerfile`：

```dockerfile
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
EXPOSE 80
EXPOSE 443

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ["MaterialManageSystem.Api/MaterialManageSystem.Api.csproj", "MaterialManageSystem.Api/"]
COPY ["MaterialManageSystem.Infrastructure/MaterialManageSystem.Infrastructure.csproj", "MaterialManageSystem.Infrastructure/"]
COPY ["MaterialManageSystem.Core/MaterialManageSystem.Core.csproj", "MaterialManageSystem.Core/"]
RUN dotnet restore "MaterialManageSystem.Api/MaterialManageSystem.Api.csproj"
COPY . .
WORKDIR "/src/MaterialManageSystem.Api"
RUN dotnet build "MaterialManageSystem.Api.csproj" -c Release -o /app/build

FROM build AS publish
RUN dotnet publish "MaterialManageSystem.Api.csproj" -c Release -o /app/publish /p:UseAppHost=false

FROM base AS final
WORKDIR /app
COPY --from=publish /app/publish .
ENTRYPOINT ["dotnet", "MaterialManageSystem.Api.dll"]
```

构建并运行Docker容器：

```bash
docker build -t materialmanageapi .
docker run -d -p 5000:80 --name materialmanageapi materialmanageapi
```

#### 方式二：使用IIS部署

1. 发布项目

```bash
cd MaterialManageSystem.Api
dotnet publish -c Release -o publish
```

2. 在IIS中创建网站
   - 打开IIS管理器
   - 创建新网站，指向 `publish` 目录
   - 设置应用程序池为 .NET 10.0

### 4.3 前端部署

```bash

```

#### 生产环境部署

1. **构建生产版本**

```bash
cd MaterialManageSystem.Vue
npm install
npm run build
```

2. **部署到Nginx**

配置Nginx (`nginx.conf`)：

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /path/to/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:5000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /hubs/ {
        proxy_pass http://localhost:5000/hubs/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
    }
}
```

## 5. 配置说明

### 5.1 JWT配置

支持两种配置方式：

**方式1：通过配置文件**（仅开发环境推荐）
```json
{
  "Jwt": {
    "Key": "YourSecureKeyHere",
    "Issuer": "MaterialManageSystem"
  }
}
```

**方式2：通过环境变量**（生产环境推荐）
```bash
# Linux/macOS
export JWT_SECRET_KEY="YourVeryLongAndSecureKey123456789"

# Windows PowerShell
$env:JWT_SECRET_KEY="YourVeryLongAndSecureKey123456789"
```

> **注意**: 环境变量优先级高于配置文件。生产环境务必使用环境变量配置密钥。

### 5.2 CORS配置

默认允许所有来源，生产环境应限制具体域名：

```csharp
app.UseCors(options => options
    .WithOrigins("https://yourdomain.com")
    .AllowAnyMethod()
    .AllowAnyHeader()
    .AllowCredentials());
```

## 8. 常见问题

### 8.1 数据库连接失败

**问题**: `Cannot connect to SQL Server`

**解决方案**:
1. 检查数据库服务器地址和端口
2. 确认SQL Server服务已启动
3. 检查防火墙设置，确保1433端口开放
4. 验证用户名和密码正确

### 8.2 JWT认证失败

**问题**: `Unauthorized` 错误

**解决方案**:
1. 检查JWT密钥是否一致
2. 确认Token是否过期
3. 检查Token格式是否正确（Bearer + Token）

### 8.3 SignalR连接失败

**问题**: WebSocket连接失败

**解决方案**:
1. 检查Nginx配置是否包含WebSocket升级
2. 确认后端服务正常运行
3. 检查防火墙是否阻止WebSocket连接

### 8.4 前端构建失败

**问题**: `npm run build` 失败

**解决方案**:
1. 更新Node.js到最新LTS版本
2. 删除 `node_modules` 并重新安装
3. 检查TypeScript语法错误

# 使用说明

## 1. 系统登录

1. 打开浏览器访问系统首页
2. 输入用户名和密码（默认：admin / Admin@123）
3. 点击登录按钮进入系统

## 2. 功能模块

### 2.1 仪表盘

展示系统概览信息：
- 库房数量统计
- 物料种类统计
- 料卷状态分布
- 快捷操作入口

### 2.2 库房管理

**功能**:
- 查看所有库房列表
- 新建库房（编码、名称、位置、备注）
- 编辑库房信息
- 删除库房（需确认）

**操作步骤**:
1. 点击左侧菜单"库房管理"
2. 点击"新建库房"按钮
3. 填写库房信息
4. 点击保存

### 2.3 物料管理

**功能**:
- 查看物料列表
- 新建物料（物料编码、名称、规格、单位、预警值）
- 编辑物料信息
- 删除物料

**预警设置**:
- 预警数量：库存低于此值时触发预警
- 预警天数：入库后超过此天数未使用触发预警

### 2.4 料卷管理

**功能**:
- 查看料卷列表（按状态筛选）
- 新建料卷（料卷编号、物料、初始数量、生产日期）
- 出库操作：将料卷从库房取出
- 上线操作：料卷投入生产使用
- 退回操作：料卷退回库房
- 报废操作：报废无法使用的料卷

**状态流转**:
```
入库 → 出库 → 上线 → 退回/报废
```

### 2.5 预警管理

**预警类型**:
- 库存不足预警
- 过期预警
- 长时间未使用预警
- 即将到期预警

**处理预警**:
1. 查看预警列表
2. 点击预警查看详情
3. 采取相应措施（补货、处理过期物料等）

## 4. 使用建议

### 4.1 数据录入规范

1. **物料编码**: 使用公司统一编码规则
2. **料卷编号**: 建议包含生产日期和批次信息
3. **预警设置**: 根据实际需求合理设置预警阈值

### 4.2 日常操作流程

**入库流程**:
1. 确认物料信息已存在
2. 创建料卷记录
3. 关联库房和仓位

**出库流程**:
1. 选择要出库的料卷
2. 填写出库人员信息
3. 确认出库数量

**上线流程**:
1. 选择已出库的料卷
2. 填写使用数量和生产信息
3. 确认上线

### 4.3 预警处理

1. 定期查看预警页面
2. 优先处理红色预警（库存不足、已过期）
3. 及时补货或处理过期物料
